# Documentation Index

Use this file as the first stop for repo documentation. Keep active docs in the
root of `docs/`; move stale, superseded, or historical material under
`docs/archive/`.

## Load First

- `PROJECT_CONTEXT.md` - stable architecture, setup contract, and decisions.
- `NOW.md` - current focus, next build steps, and short-term working memory.
- `agent-handoff.md` - active cross-agent handoff ledger.
- `SESSION_NOTES.md` - append-only session history.

## Active References

- `phased-plan.md` - canonical roadmap and completed phase state.
- `deploy-checklist.md` - production release checklist.
- `DM-rules.md` - deterministic adjudication and narration guardrails.
- `multiplayer-design.md` - co-op session/state architecture.
- `visual-multiplayer-phase0.md` - visual shell and asset/UI plan.
- `visual-asset-generation.md` - controlled generated-asset workflow.
- `agent-crossover-contract.md` - Codex/Claude/Antigravity ownership and review rules.
- `AGENT_SESSION_PROTOCOL.md` - local memory-file session workflow.

## Archive

- `archive/historical/` - superseded implementation notes and old branch docs.
- `archive/planning/` - completed or folded-in sprint/roadmap planning docs.
- `archive/process/` - template/process scaffolding not needed for normal work.

Archived docs are retained for context only. Prefer the active references above
when planning or implementing new work.
